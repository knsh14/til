# arxiv 新着論文ダイジェスト

arxiv のその日の新着論文をカテゴリ別に取得し、各論文を日本語で要約して Markdown ファイルとしてリポジトリに保存する。

## 設定ファイル

`arxiv-digest/categories.json` に対象カテゴリを記載する。ファイルが存在しない場合はスクリプト内蔵のデフォルト値（下記）が使われる。

```json
{
  "top_level": ["cs", "math", "physics", "stat", "eess", "q-bio", "q-fin", "econ"],
  "subcategories": ["cs.CV", "cs.RO", "cs.AI", "cs.LG", "cs.SY", "stat.ML", "stat.AP", "stat.ME"],
  "labels": { "cs": "Computer Science", "cs.CV": "CS - Computer Vision and Pattern Recognition", ... }
}
```

各カテゴリ最大10件の新着論文を取得する。トップレベルとサブカテゴリで論文が重複する場合があるが、それぞれの視点での一覧として両方残す。

## ワークフロー

### Step 1: 論文データの取得

スキルに同梱されている `scripts/fetch_arxiv.py` を実行する。

```bash
uv run python <skill-dir>/scripts/fetch_arxiv.py
```

- 標準出力に JSON が出力される（カテゴリごとにグループ化された論文データ）
- 各論文には `title`（英語原題）、`abstract`（英語要旨）、`link`、`authors`、`category` が含まれる
- arxiv のレート制限に従い、リクエスト間に3秒の待機が入るため、全カテゴリの取得に約1分かかる
- 取得失敗時は最大3回リトライする（指数バックオフ）

> **subagent を使う場合は `model: "opus"` を指定すること。**

### Step 2: サマリー生成・構成・保存

取得した JSON を読み、以下を一括で生成する:

1. **各論文の日本語サマリー**:
   - **タイトル（日本語訳）** — 原題の意味を正確に伝える自然な日本語
   - **要約** — 2〜3文で論文の内容・手法・結果を簡潔にまとめる
   - **新規性** — 1〜2文で、この論文が既存研究と比べてどこが新しいのかを明確に述べる

2. **ハイライトセクション** — 全カテゴリから注目論文を3〜5件選定。選定基準:
   - 複数カテゴリにまたがるインパクトがある
   - 既存の主要手法を大きく上回る結果を報告している
   - 新しい問題設定やアプローチを提示している
   - 自動運転に関連する実応用への示唆がある

3. **カテゴリごとの導入文** — 新着論文の全体的な傾向を1〜2文で記載

4. **整合性チェック**:
   - 同じ論文が複数カテゴリに出現する場合、要約内容に矛盾がないこと
   - 用語の表記が統一されていること（例:「大規模言語モデル」と「LLM」が混在しない）
   - 要約の粒度がカテゴリ間で偏っていないこと

要約の注意点:
- abstract の内容に忠実に書く。推測や外部知識で補わない
- 専門用語はそのまま使ってよいが、略語は初出時に正式名称を併記する
- 「〜を提案した」「〜を示した」のように、論文が何をしたのかを主語明確に書く

### Step 3: Markdown ファイルの保存

以下のフォーマットで Markdown ファイルを生成する:

```markdown
# arxiv 新着論文ダイジェスト — YYYY-MM-DD

## 本日の注目論文

1. **[{日本語タイトル}]({link})** ({カテゴリ}) — {選定理由を含む一文紹介}
2. ...

---

## {カテゴリ表示名} ({カテゴリID})

> {カテゴリの傾向を示す導入文}

### 1. [{日本語タイトル}]({arxiv link})
**要約:** {2〜3文の要約}

**新規性:** {1〜2文の新規性}

---

### 2. ...

## {次のカテゴリ} ...
```

保存先: `arxiv/YYYY-MM-DD.md`
- `arxiv/` ディレクトリが存在しない場合は作成する
- 同名ファイルが既にある場合は上書きする

### Step 4: 完了報告

生成が完了したら、以下を報告する:
- 保存したファイルパス
- カテゴリごとの論文数
- 取得できなかったカテゴリがあればその旨

## エラーハンドリング

- 特定のカテゴリの取得に失敗した場合（3回リトライ後も失敗）、そのカテゴリをスキップして他を続行する
- 新着論文が0件のカテゴリは「本日の新着論文はありません」と記載する
- スクリプト自体が失敗した場合は、エラー内容をユーザーに伝える
