# arxiv 新着論文ダイジェスト

arxiv のその日の新着論文をカテゴリ別に取得し、各論文を日本語で要約して Markdown ファイルとしてリポジトリに保存する。

## 対象カテゴリ

### トップレベル（各10件）
cs, math, physics, stat, eess, q-bio, q-fin, econ

### サブカテゴリ（各10件） — 自動運転関連
**CS:** cs.CV, cs.RO, cs.AI, cs.LG, cs.SY
**Stat:** stat.ML, stat.AP, stat.ME

トップレベルとサブカテゴリで論文が重複する場合があるが、それぞれの視点での一覧として両方残す。

## ワークフロー

### Step 1: 論文データの取得

スキルに同梱されている `scripts/fetch_arxiv.py` を実行する。

```bash
python <skill-dir>/scripts/fetch_arxiv.py
```

- 標準出力に JSON が出力される（カテゴリごとにグループ化された論文データ）
- 各論文には `title`（英語原題）、`abstract`（英語要旨）、`link`、`authors`、`category` が含まれる
- arxiv のレート制限に従い、リクエスト間に3秒の待機が入るため、全カテゴリの取得に約1分かかる

### Step 2: 日本語サマリーの生成

取得した JSON を読み、各論文について以下の3点を日本語で作成する：

1. **タイトル（日本語訳）** — 原題の意味を正確に伝える自然な日本語
2. **要約** — 2〜3文で論文の内容・手法・結果を簡潔にまとめる
3. **新規性** — 1〜2文で、この論文が既存研究と比べてどこが新しいのかを明確に述べる

要約を作る際の注意点：
- abstract の内容に忠実に書く。推測や外部知識で補わない
- 専門用語はそのまま使ってよいが、略語は初出時に正式名称を併記する
- 「〜を提案した」「〜を示した」のように、論文が何をしたのかを主語明確に書く

### Step 3: 文章の構成と編集

個別のサマリーを生成した後、ダイジェスト全体として読みやすくなるよう構成を整える。

1. **ハイライトセクションの作成** — ファイル冒頭に「本日の注目論文」として、全カテゴリの中から特に注目すべき論文を3〜5件ピックアップし、選定理由とともに簡潔に紹介する。選定基準は以下のいずれか：
   - 複数カテゴリにまたがるインパクトがある
   - 既存の主要手法を大きく上回る結果を報告している
   - 新しい問題設定やアプローチを提示している
   - 自動運転に関連する実応用への示唆がある

2. **カテゴリ間の整合性チェック** — 全カテゴリを通して以下を確認する：
   - 同じ論文が複数カテゴリに出現する場合、要約の内容に矛盾がないこと
   - 用語の表記が統一されていること（例：「大規模言語モデル」と「LLM」が混在しない）
   - 要約の粒度がカテゴリ間で偏っていないこと

3. **カテゴリごとの導入文** — 各カテゴリセクションの冒頭に、そのカテゴリの新着論文の全体的な傾向を1〜2文で記載する（例：「本日の cs.CV は、3D物体検出と動画理解に関する論文が多い」）

### Step 4: Markdown ファイルの生成と保存

以下のフォーマットで Markdown ファイルを生成する：

```markdown
# arxiv 新着論文ダイジェスト — YYYY-MM-DD

## 本日の注目論文

1. **[{日本語タイトル}]({link})** ({カテゴリ}) — {選定理由を含む一文紹介}
2. ...

---

## {カテゴリ表示名} ({カテゴリID})

> {カテゴリの傾向を示す導入文}

### 1. [{日本語タイトル}]({arxiv link})
**要約:** {2〜3文の要約}

**新規性:** {1〜2文の新規性}

---

### 2. ...

## {次のカテゴリ} ...
```

保存先: リポジトリの `arxiv/YYYY-MM-DD.md`
- `arxiv/` ディレクトリが存在しない場合は作成する
- 同名ファイルが既にある場合は上書きする

### Step 5: 完了報告

生成が完了したら、以下を報告する：
- 保存したファイルパス
- カテゴリごとの論文数
- 取得できなかったカテゴリがあればその旨

## エラーハンドリング

- 特定のカテゴリの取得に失敗した場合、そのカテゴリをスキップして他を続行する
- 新着論文が0件のカテゴリは「本日の新着論文はありません」と記載する
- スクリプト自体が失敗した場合は、エラー内容をユーザーに伝える
