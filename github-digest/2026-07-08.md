### ![PR](icons/git-pull-request.svg) [manager: catch SIGTERM/SIGHUP in github_runner.sh wrapper](https://github.com/sunnypilot/sunnypilot/pull/1872) `#1872`
**ステータス:** Open | **ラベル:** なし

**概要:** `system/manager/github_runner.sh` の trap が SIGINT / SIGKILL / EXIT しか監視していない問題を修正する。SIGKILL はトラップ不可のため列挙は無意味で、SIGTERM が欠落している。manager は SIGINT の後、タイムアウトで SIGTERM へエスカレートするため、SIGTERM 着信時に trap が実行されず `systemctl stop actions.runner.sunnypilot.<host>` が呼ばれず、actions-runner の systemd サービス（dotnet 製 Runner.Listener / Runner.Worker）が残留する。残った runner が plannerd / modeld と CPU を奪い合い、走行中に commIssue / softDisable による "TAKE CONTROL IMMEDIATELY" が断続的に発生したと報告している。SIGTERM / SIGHUP を捕捉し SIGKILL を除外することで、正常終了シグナルで確実に停止フックが走るようにする。

**直近のアクティビティ:** copilot ボットがレビュー概要を投稿し、シグナルトラップの SIGTERM / SIGHUP への拡張と SIGKILL 除外を要約している。ユーザーからのコメントはなし。

**変更ファイル:** 1 ファイル・+7/-2。`system/manager/github_runner.sh` のみ。ループや start パスの挙動、車両・安全コードには変更なしとしている。

**次のアクション:** メンテナによるレビューとマージ判断。
