### ![Issue](icons/issue-opened.svg) [`package.path` includes `./?.lua` allowing arbitrary code execution from current directory](https://github.com/neovim/neovim/issues/38966) `#38966`
**ステータス:** Open | **ラベル:** security, lua

**概要:** Neovim のデフォルト `package.path` に `./?.lua` が最初のエントリとして含まれており、カレントディレクトリにある Lua ファイルが `require()` で読み込まれてしまうセキュリティ脆弱性。悪意のあるファイルがプラグインモジュールを乗っ取る可能性がある。

**直近のアクティビティ:** bfredl は「これは Lua、Python、cmd.exe 等で数十年来の正常な動作」とコメント。clason は `nvim -l` でのスタンドアロン Lua ランタイム使用時に `./?.lua` が必要な場合があるため、単純に削除するのは困難と指摘。superatomic はサードパーティプラグインが `pcall(require, ...)` で選択的にチェックする場合に脆弱になると指摘。

**次のアクション:** 議論継続中。`nvim -l` との互換性を考慮した解決策の検討が必要
