4. **[modem: register cellular DNS with systemd-resolved](https://github.com/commaai/openpilot/pull/37955)** (commaai/openpilot) — 自前の `modem.py` 全面移行（#37811）後、セルラー DNS が NetworkManager 経由で設定されない取りこぼしを修正。Wi-Fi 切断時にインターネットへ到達できない既知の不具合を解消する。リプレースバウンティ完了直後の代表的なリグレッションフォローアップ。
5. **[feat(tui): apply terminfo overrides from $NVIM_TERMINFO](https://github.com/neovim/neovim/pull/39555)** (neovim/neovim) — 環境変数 `$NVIM_TERMINFO` で terminfo を JSON 上書きできる機能。Windows での terminfo 利用問題（#37274）への対応であり、Neovim 全体の terminfo 依存を縮小していくロードマップに沿う一歩。
6. **[job control: vim.SystemObj:kill() leaves orphans](https://github.com/neovim/neovim/issues/39552)** (neovim/neovim) — `vim.system` で起動した子プロセスを `:kill()` してもプロセスグループへ信号が届かず、孫プロセスが孤児化する問題。`jobstart`/`jobstop` には存在しない 5 件目の差分で、`vim.system` と `jobstart` の二重ジョブ制御層を一本化すべきというアーキテクチャ議論に発展。短期対処 PR は [#39562](https://github.com/neovim/neovim/pull/39562)。
